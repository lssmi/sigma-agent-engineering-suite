production-readiness-report + 三组看板

⚑ 关键拦截 1 · 静默逃逸被独立审查当场打回

生成者初版贴最高 RPN → 「根因=刀具磨损，建议换刀」(置信 0.86)。审查者读独立数据源反驳：刀具寿命内(520/800件)、OD 漂移与「件数」弱相关、与「来料批次切换日」强相关 → REJECT 打回。裸模型会把"换刀"当结论交付——这是第3章「静默逃逸」被挡下的一刻。

⚑ 关键拦截 2 · 不可逆动作按可逆性挂人工（与置信度解耦）

冻结物料 / 发召回函 = 不可逆 → 即便 Agent 报 0.97 高置信，照样挂人工；刀补 +0.015 = 可逆 → 放行。「能撤放手做、不能撤先问，高置信但撤不回照样先问」。

↻ 慢外环咬合 · 三周后的判例飞轮（系统真正的回报）

看板报警：夜班根因返工率系统性高于日班(跳出控制带,非噪声) → DMAIC 一圈：真因不在模型、在第③步取数：夜班来料批次字段经常为空（正是 P3 被 dropna 掉的那 14 条！）→ Improve 加约 50 行「来料批次完整性」前置校验 → Control 固化成永久门禁 + 进下一版 CTQ 红线。同一类失败再也过不来。
