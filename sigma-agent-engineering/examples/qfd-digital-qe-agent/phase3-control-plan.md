> 实现者：Claude（codex 评审）。把 Phase 二的子系统能力落到可执行/可验证/可运维的过程控制；承接各 How 目标值作验收门。对应主 skill 的 P7–P11（测量总线/护栏/DMAIC/生产准入）。

## 2. 上线控制计划（五步生产准入，对应主 skill P11 / 书第22章 22.2）
1. **SOP 就绪性审查**：验收标准(各 How 目标值)清晰且独立于实现（oracle 先于实现）。
2. **沙盒压力测试**：功能 / 对抗(注入) / 韧性三类——对应 S3 逃逸、S8 红队、S5 脏数据。
3. **影子运行**(1–2 周，输出不执行)：真实分布上测各 How 达标率（正是 8D run 里抓出"夜班来料缺失"的那一步）。
4. **灰度放量**：5%→20%→…，能力门先于流量门，任一档不达标即回退。
5. **全量发布 + 30 天观察期**：跌破红线一键回退/熔断。

## 3. 变更控制 & 数据治理
- **模型/知识库变更控制**：模型路由、判例库、FMEA 库变更走版本化 + 回归测试（碰掉验收门即报红）。
- **验证集/仿真异常案例**：维护反例驱动评估集（含历史真实犯错固化为永久反例）；定期保鲜（旧题库=没校准的卡尺）。
- **8D 报告审查规则**：Schema 硬门 + 人工签核；事实/推断/假设/建议分离留痕。
- **记忆代谢**：ECN 事件驱动 + TTL 兜底（书表15-2 口径），代谢率 5–15%/月上看板。

## 4. 控制点站位（对应 IQC/IPQC/OQC，校验离起因越近越好）
- **IQC 入口**：S1 数据对齐门、S2 异常输入校验、外部内容裹 untrusted。
- **IPQC 工序间**：S3 每步独立校验、S5 证据覆盖门、S7 不可逆动作前人工门。
- **OQC 出口**：S6 Schema 合规门、零写回/外发白名单、S8 越权拦截。

## 5. 与主 skill 两速控制收口（R≈pⁿ）
- 快内环逐步门禁（IQC/IPQC/OQC + 可逆门）压方差、抬地板；
- 慢外环（S9 判例飞轮 + 独立测量总线 S2/S8）把每次逃逸/返工固化成永久门禁与下一版 CTQ 红线；
- 控制计划的终极验收 = 端到端 **R = ∏ p_i** 达到容差带（`pⁿ` 仅便于建立直觉的乐观基线，**不可简化成纯 pⁿ**；制造质量签署＝端到端逃逸率 ≤0.1%）。

## 6. 评审后补正（Codex round-3 评审采纳）
- **补回 S4 知识检索控制计划**（见 §1 表已加行）。
- **量化模糊门**：S2 漏报率 <2%（控制带派生）；S8 越权拦截率≥99%（挂 HOW-08 辅助）；S9 验收门挂 **HOW-11**（判例复用≥40%）。
- **全局一键回退对象**：模型版本 / 知识库(FMEA·判例)版本 / 规则版本 / 流量开关 / 数据口径版本——任一可独立回滚。
- **灰度晋级/回退条件（非凭感觉）**：每档须连续 ≥3 天 How 门达标 + 无 P0/P1 逃逸 + 人工接管率不超阈值，才晋级；任一破线即回退该档。
- **评估集污染防护**：训练/调参不得触碰 OQC 留出的验收/反例集（数据集版本冻结）。
- **容差带表述修正**：第 5 节"制造质量签署 ≤0.1%"指**端到端逃逸率 ≤0.1%**。
