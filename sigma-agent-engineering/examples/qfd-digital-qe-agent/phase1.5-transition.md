## 3. △ 情境放大风险点

| CTQ×节点 | 为何在此放大 | 由谁兜 |
|---|---|---|
| CTQ-01 × D3 临时围堵 | 围堵动作直接影响生产连续性，过度冻结或停线会把诊断问题放大成产能损失。 | 人工门兜底：冻结物料、停线必须由授权现场负责人确认。 |
| CTQ-05 × D5 永久纠正 | 永久纠正可能牵涉工艺参数、供应商处置、客户沟通，动作影响不可轻易回滚。 | 护栏+人工门兜底：按可逆性识别，不因置信度高而跳过确认；对外发函必须人工批准。 |
| CTQ-06 × D3 临时围堵 | 若系统在高不确定性下仍自动放行，会形成静默逃逸；若误判低风险，缺陷可能继续流出。 | 护栏兜底：不确定/证据不足时宁可喊停转人工；高风险批次进入人工复核队列。 |
| CTQ-09 × D4 根因 | D4需要跨多系统追溯，查询链路最长、计算最重，最容易拖慢现场节奏。 | 系统护栏兜底：先给Top-N诊断方向与证据摘要，深查异步补全；性能超时转人工研判。 |
| CTQ-02 × D4 根因 | 根因结论若只靠相关性或历史相似案例，容易在复核时被推翻。 | 独立审查兜底：根因必须挂接证据链覆盖率、反证记录与D6验证结果。 |

| 工作情境 | CTQ归入 |
|---|---|
| 检测 | CTQ-06 该报警不漏报、不静默逃逸；CTQ-08 过程漂移早期预警、提前量足 |
| 定位 | CTQ-01 不停产/最短停机下给可行动诊断方向；CTQ-02 根因有证据支撑、可追溯、经得起复核；CTQ-03 跨系统数据自动对齐、免人工穿梭对数 |
| 验证 | CTQ-02 根因可复核；CTQ-06 不静默逃逸；CTQ-10 投入产出可量化、价值可追溯 |

| 纠正预防 | CTQ-07 历史教训可复用、同类问题不二次发生；CTQ-08 漂移预警进入控制计划；CTQ-10 措施价值可追溯 |
| 报告合规 | CTQ-04 8D报告合规完整、可下发、可挂签；CTQ-05 关案人工门 |

## 5. 对 Phase 二的输入

| 8D节点 | 需要的子系统能力 |
|---|---|
| D0 征兆/触发 | 多源异常采集；SPC/设备/环境漂移检测；报警分级；静默逃逸监控；快速Top-N方向生成 |
| D1 组队 | 角色/责任人推荐；签核链识别；问题类型到专家/部门映射；历史相似案例索引 |
| D2 问题描述 | MES/ERP/SPC/设备日志字段自动对齐；批次/工单/设备/时间窗结构化；合规字段完整性校验 |
| D3 临时围堵 | 批次影响范围计算；围堵建议生成；冻结物料/停线人工门；高不确定性转人工策略 |
| D4 根因 | 跨系统证据链构建；根因Top-N排序；FMEA/历史8D检索；证据覆盖率与可追溯链路 |
| D5 永久纠正 | 措施库匹配；纠正措施影响评估；对外发函人工门；措施成本/收益记录 |
| D6 验证 | 措施有效性验证；复发/漏检监控；验证样本与时间窗管理；独立复核记录 |
| D7 防再发 | FMEA更新建议；控制计划/阈值更新；判例沉淀；同类问题复用与复发监控 |
| D8 结案表彰 | 8D报告生成；合规字段完整性检查；关案人工门；收益归档；知识库入库 |
