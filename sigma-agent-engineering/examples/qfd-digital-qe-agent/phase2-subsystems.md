## 3. 优先开发序与理由
- **P1（先做）**：S1 数据连接（无底座一切空）、S3 诊断推理+独立审查（瓶颈步+承接最多一票否决/最高加权243）、S7 人审控制（不可逆安全硬门）。
- **P2**：S2 异常检测、S5 证据编排、S6 文档生成（合规可下发）。
- **P3**：S4 知识检索、S8 权限审计。
- **P4**：S9 持续学习（判例飞轮，越用越好、慢热）。

## 3b. 评审后补正（Codex round-2 评审采纳）
- **HOW-09 工时降低≥60%**：横切结果指标，由 S1(免对数)+S3(自动诊断)+S6(自动报告) 共同承接，非单一子系统（补回上游 HOW-09 映射）。
- **目标值具体化**：S1 HOW-01 取数 p95<8s；S2 漂移提前量≥3 批次(待试点校准)；S4 知识命中率≥80%；S8 trace 留存=运营 90 天/审批 3 年；S9 代谢率 5–15%/**月**。
- **安全硬门 vs 加权排序解耦（写成规则）**：S3 独立审查 / S6 Schema 门 / S7 人审门的**最小能力不得因加权分低被降级**；S8 权限审计拆**最小审计子集提到 P2**（与 S7 人审门闭环），完整审计留 P3。

## 4. 与主 skill 架构的对应（沿 R≈pⁿ 两速控制）
- 快内环(P1–P6)：S1=接入层(MCP)、S5=信息/证据控制面、S3沙盒计算+推理、S6+S7=三道防线护栏(Schema门+可逆门)。
- 慢外环(P7–P11)：S2/S8=独立测量与审计、S9=DMAIC 判例飞轮。
- 三手段：S1/S5 抬 p(喂干净输入)、S3 抬 p(降方差)+独立审查封顶(Safety-I)、S7 封顶(不可逆兜底)。
