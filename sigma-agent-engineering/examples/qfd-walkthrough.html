↯ 效果：四步四个真错，双向都中招

Claude 的算术错(285≠243)、结构遗漏(漏 S4)、单位/对象污染，Codex 的漏列——各自盯不到自己的盲区，跨模型互审一抓一个准。这就是「运动员≠裁判」的硬价值：同源自审(同一个脑袋既写又审)发现不了这些；换一个异构模型来审才行。

🐛 过程中的工程教训：别让 Codex 卡死

  • 进程泄漏：反复 launch+kill 顶层进程，Codex 派生的 node/darwin 子进程没回收，堆到 59 个拖垮连接 = 连接挂死的本地根因。
  • 看门狗启动器 cdx.sh：90s 无 banner → 判连接挂、秒杀 / 封顶 7min / 跑完 pkill reap 子进程。
  • 实测：连接挂从 15–25min 闷死 → CONNECT_STALL@90s 快杀、残留进程=0；连上的 DONE@60–90s。
  • 配方：重生成(implement)易中途 stall，low effort 生成快、在 stall 窗口前跑完更易成；-o 可能在 kill 后才落盘(别误判失败)，输出也在 session rollout 可恢复。
