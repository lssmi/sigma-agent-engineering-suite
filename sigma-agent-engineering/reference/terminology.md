## B.1　Sigma Engineering 新术语（本书自造 / 重定义）

每条 = 一句话定义 + 首现章。来源：构建期蓝图 `blueprint.terminology_canon` 17 条（本书核心概念；蓝图未随仓发布，内容已固化进本表）。**注：并非全部都是附录 B.1 词条**——如"三层控制面"属附录 B.2 通用术语，"三道防线 / 致命三联 / 双 LLM 隔离 / 算力归代码"为正文概念（首现章见各条"首现"列）；本表按概念整理，非附录 B.1 逐条复制。

### R≈pⁿ（主线公式）
任务整体可靠度 R 约等于单步可靠度 p 的步数 N 次方——可靠性是**乘出来的，不是加出来的**。真实写法是条件概率连乘 `R = ∏ p_i`（`p_i = P(step_i 正确 | 上游状态 / 上下文长度 / 噪声 / 既有错误)`）；`pⁿ` 是便于建立直觉的乐观基线，真实衰减往往更陡（self-conditioning 式误差传播）。基准直觉：`0.95²⁰ ≈ 0.36`。
**首现：** 总论 0.1。已对 04 真文件核实。

### 复合主导（Compounding Dominance）
多步不确定过程里，误差**相乘而非相加**，指数项支配一切：十个 95% 连乘只剩约 60%，且崩塌几乎没有预兆。复杂任务真正的敌人不是「模型不够聪明」，而是误差在链条上复利。
**首现：** 总论 0.2。

### 蓄水池机制
复合误差的直观画面：每步小误差不会各自消散，而是像水滴逐步累积为**上下文污染（context pollution）**与**状态漂移（state drift）**；任务崩在「池满溢出」而非「某滴特别大」——所以要盯水位（累积量），不只盯水滴（单步成功率）。
**首现：** 总论 0.2。

### 三手段
对 `R≈pⁿ` 你只能做三件事：
① **抬高单步可靠度 base p**——关键是**降单步方差**而非抬均值（时而 A 时而 D 的步骤比稳定 B+ 更危险）；
② **缩短步数 N**——删不增值的随机步，把确定逻辑剥给代码；
③ **给失败兜底封顶（⊓）**——让失败被及时发现、拦下、安全降级，含 Safety-I + Safety-II，前置条件是**状态外置 + 重试幂等**。
**首现：** 总论 0.3。

### 快内环（执行）
两速系统中毫秒到秒尺度的执行环：感知 → 规划 → 调工具 → 强制校验，每一步通过门禁才放行。它**不学习、不反思**，只压住单次任务每一步的方差、不让误差当场复合——即「驾驭工程（Harness）」本身。
**首现：** 总论 0.4。

### 慢外环（改进）
跨周期尺度的改进环：把快内环跑出的真实失败沿 **DMAIC**（界定—测量—分析—改进—控制）走完一整圈，固化为一道**永久门禁**。它才是 Agent「越用越可靠」的来源；护栏 / 人工复核 / 记忆代谢 / 可观测性 / 成本工程 / 规模化准入拆到底都是慢外环零件。
**首现：** 总论 0.4。

### 独立测量总线
耦合快慢两环的那条线：采集每条 trace、做评估、换算成逃逸率，慢外环以这些真实测量为**唯一改进依据**。不可妥协的铁律：**测量必须独立于被测对象**——让 Agent 给自己诊断打分等于自证清白。电商「安慰剂对照组」是其落地。
**首现：** 总论 0.4 · 第 18 章。

### 判例飞轮 · C→D
从 Control 折回 Define 的那根回流弧：把每次人工拍板 / 熔断黑匣子 / 近失固化为下一版 Define 和 CTQ 红线——一条失败被固化成谁都绕不过去的门、再也无法第二次发生。是机器里最有价值的一根线，企业「带不走的资产」在数学上的样子全长在它上面。
**首现：** 总论 0.4 · 第 21 章。

### 逃逸率（escape-ppm）/ 静默逃逸
**静默逃逸**：不报警的错最危险（出错与报错被解耦），如把该挂起复检的料判「可放行」且不触发任何报警。**逃逸率（escape-ppm）**：本该被拦住却漏到下游、尤其是被静默放行的错误比例，是测量总线把 trace 换算成的核心 KPI。误差预算就是给逃逸率设上限（如千分之三）当可花的预算。Sigma Engineering 的核心是**杀掉静默逃逸**，而非追求偶尔惊艳。
**首现：** 总论 0.4 · 第 18–19 章。

### 三层控制面
模型外三层**物理隔离**（不靠模型配合、靠外部代码强制）：
- **指令面（Instruction）**：把散文升级为 API 契约 / 强制 Schema，只能翻转指定状态字段；
- **信息面（Information）**：Need-to-Know 按需供给 + 动态折叠 + 定界包裹不可信内容；
- **运行治理面（驾驭工程 Harness）**：状态机 → 权限隔离 → 强制校验熔断 → 人工审批四步闭环。
**首现：** 第 4 章。

### 三道防线
治理篇第 13 / 14 / 15 章体系（章节标题原文用「第 N 层防线」）：
- **第一道**＝结构化护栏 **Safety-I**（IQC / IPQC / OQC，堵列得出的失败）——第 13 章；
- **第二道**＝**人工复核可逆门**（按可逆性而非置信度，接护栏接不住的体面失败）——第 14 章；
- **第三道**＝**组织级记忆 / 权限隔离 / 知识代谢**——第 15 章。
**首现：** 第 13 / 14 / 15 章。已对真文件核实章节标题。

### Safety-I / Safety-II
- **Safety-I**＝处理**可预见**失效：预设拦截规则（Schema 校验 / 熔断 / 物料冻结超量拦截），把列得出的失败提前写成门；
- **Safety-II**＝应对**不可预见**异常：靠优雅降级（graceful degradation）——触界给带「数据异常待人工确认」标记的部分结果、退回只读、安全停机转人，接列不出的意外。
两者缺一不可，前置条件是状态外置 + 重试幂等。
**首现：** 总论 0.3。

### 致命三联（lethal trifecta）
Simon Willison 2025 提出：同一 Agent 同时握有 **① 读私有数据 + ② 接触不可信外部内容 + ③ 对外发送通道** ＝ 完美数据外泄管道。属**正交轴 II（对抗性）**，非 `pⁿ` 随机误差。解法：结构上拆掉至少一环；号称「拦九成五」的护栏对安全仍不够。
**首现：** 第 13 章。已对 ch13(file17) 核实。

### 双 LLM 隔离（dual-LLM）
一个**有权限的主模型**只处理可信指令、负责规划；另一个**被隔离、不带工具的模型**专读不可信内容，产出只能是经严格校验的结构化结果或符号变量，**绝不以原文 / 指令形态回流主模型**，两者由确定性代码居中传递。更进一步的 Conseca 框架把策略生成隔离到可信侧 + 独立确定性「策略执行器」逐条裁决、不经大模型推理，对注入天然免疫。
**首现：** 第 13 章。已对 ch13 核实。**来源：** Willison, S. (2023) *The Dual LLM Pattern*（与致命三联 Willison **2025** 为两条不同年份来源，勿混）。

### 正交轴 II · 信任与遏制
独立于 `pⁿ` 的第二根轴：对付的是**对抗性**（提示词注入 / 越权）而非随机误差——把对抗性硬塞进 `pⁿ` 是错的。三原则：
① 外部内容当**数据**而非指令；
② 别让一个 Agent 同时握私有数据 + 读不可信内容 + 对外发送（致命三联）；
③ 最小权限并在**网关上机械执行**。
**首现：** 总论 0.5 · 第 13 章。

### 算力归代码，解释归模型
凡有确定答案的计算（CpK 等有唯一公式的指标）绝不让模型猜，让它写代码给出可复验的确定性结果；模型只保留「读懂数字、给出解释」那一半。即 TRIZ 空间分离，同时抬 p（方差归零）+ 缩 N。
**首现：** 第 8 章（贯穿总论 0.3 / 第 1 / 4 章三手段）。

> **解释半边的两个失败模式（本地 lesson·dogfooding）**：「解释归模型」只写了赢面，实战里这半边有两种退化——
> ① **空壳化**：LLM 其实没接进路径，规则模板静默冒充「AI 解释」。判据＝解释路径是否真有 LLM 客户端、`*_LLM` 开关是否 default `off` 却对外称 AI。修法＝产品表面诚实标注「引擎结论·规则生成」，不冒充模型洞察。
> ② **越权改数**：LLM 真接入后改写 / 编造引擎已算定的统计量直达用户（Cpk 1.33→1.3 漂移、编造 9.99）。修法＝解释层数值引用门（见 `agent-guardrails-config`「数值引用门」），oracle=真算产物而非另一个 LLM（Generator≠Critic，把第18章独立测量精神落到快内环）。
> 两失败模式共用一条铁律：**解释可以归模型，但数值的真伪权归代码**。`tablygo` 实证：其「AI 解释」一度是规则模板（空壳化）、且 LLM 接入点零校验（越权改数风险），整改为诚实标签 + 数值引用门。引用为本地实例、非本套件 skill。

### 抬地板不抬天花板
Sigma Engineering 的诚实定位：模型能力上限（天花板）随规模与推理计算进步上移，本书**不抢这件事**；它专做另一件没人替你做的事——抬高可靠性下限（地板）、消除静默逃逸。天花板越高、地板托得住，**二者相乘**才是真正可交付的能力。
**首现：** 总论 0.6。

### 可逆门 / 控制档位
- **可逆门**：人工门 = f(可逆性)，与置信度**彻底解耦**——能撤的放手做、不能撤的先问（高置信但撤不回照样先问）。
- **控制档位**：按任务复杂度配（查表算式 → 确定性代码 / 有套路 → 指令面契约 + 轻校验 / 真复杂 → 完整运行治理面 / 不可逆 → 强制人工门），遵循**必要多样性**（控制的花样 ≥ 任务的花样）。
**首现：** 总论 0.4 · 第 4–5 章。

## B.2　通用术语（行业既有，统一口径）

来源：附录 B.2 逐条。给跨背景读者一个统一口径，避免与 B.1 新术语混淆。

| 术语 | 一句话定义 | 首现 |
|---|---|---|
| **驾驭工程 Harness** | 把模型包裹进一套受控的执行结构（动作空间 / 工具 / 观测格式 / 校验门），让概率性的智力变成可交付能力；即本书的**快内环**。 | 总论 0.4 · 第 4 章 |
| **三层控制面** | 把 Agent 治理拆成信息控制（喂什么）、指令控制（输出契约 / Schema / 工具协议）、运行治理（状态机 / 权限 / 熔断 / 审批）三层，对应快内环的装配骨架。 | 第 4 章 |
| **MCP（模型上下文协议）** | Model Context Protocol，把 Agent 与外部工具 / 数据源之间的接入标准化的协议，终结「每个集成各写一套」的集成地狱。 | 第 6 章 |
| **RAG（检索增强生成）** | Retrieval-Augmented Generation，先检索再生成；Agentic RAG 进一步把检索做成可由 Agent 主动编排、过滤、迭代的**信息控制面**。 | 第 7 章 |
| **状态机** | 把任务显式建模为有限的状态与允许的转移，让 Agent 按确定路径推进、而非在自由文本里漂；编排篇的核心控制结构。 | 第 10 章 |
| **幂等 idempotent** | 同一操作执行一次与多次效果相同；重试安全的前提——按两次冻结物料必须等于按一次，否则重试风暴会重复发单。 | 总论 0.3 · 第 8 章 |
| **过程能力 Cp / Cpk** | 衡量过程输出分布相对规格限有多大余量的指数；本书借它把「业务能容忍多少瑕疵」翻译成对 `pⁿ` 曲线落点的要求。 | 第 3 章 |
| **MSA / GR&R（测量系统分析）** | 评估「量具本身可不可信」的方法；对应「量的人不能是被量的人」——测量系统的独立性必须先被验证。 | 第 18 章 |
| **FMEA（失效模式与影响分析）** | 系统性列出「会怎么坏、坏了多严重、多大概率、能否被发现」的预防工具；**Safety-I 的经典武器**。 | 第 13 章 · 第 17 章 |
| **误差预算 error budget** | 把可用的「出错额度」当预算来花：预算之上别再镀金，把劲使到真瓶颈上；来自 SRE，对应可靠性记账。 | 第 17 章 · 第 20 章 |
| **8D（D1–D8）** | 制造业经典的八步纠正与预防流程；本书主案例的诊断—整改链条，全程约二十来步具体动作。 | 总论 · 第 1 章 |
| **DMAIC** | 六西格玛改进既有过程的五阶段：界定 / 测量 / 分析 / 改进 / 控制；本书**慢外环**的骨架。 | 总论 0.4 · 第三篇 |
| **CDOC** | DFSS 设计全新过程的四阶段：概念定义 / 设计 / 优化 / **能力验证**（Concept / Design / Optimize / **Capability**）；本书「出生即高可靠」的骨架。详见下方专条。 | 第 11 章 · 第 22 章 |
| **HITL（人在回路）** | Human-in-the-Loop，在 Agent 决策链上设人工审批 / 复核节点；本书要求**按可逆性、而非按置信度**放这些节点。 | 第 14 章 |
| **trace** | 一次 Agent 运行留下的可追溯执行记录（输入 / 步骤 / 工具调用 / 校验结果）；独立测量总线的原材料。 | 总论 0.4 · 第 19 章 |
| **灰度发布 canary** | 把流量按比例一档档放给新版本、任一档不达标即回退的上线方式；本书把它读成一条可靠性增长曲线。 | 第 22 章 |

### CDOC 专条（归属与末阶段写法，回核第 11 章原文）

> **CDOC = Concept – Design – Optimize – Capability**（概念 / 设计 / 优化 / 能力验证），是面向六西格玛设计（**DFSS, Design for Six Sigma**）的一种**经典四阶段实现路线**。

- **归属（非本书自创）**：由六西格玛部署先驱、**SBTI（Sigma Breakthrough Technologies, Inc.）**创始人 **Stephen A. Zinkgraf 博士**及其团队（连同姊妹路线 **CDOV**）发展并体系化；其权威阐述见 SBTI 高管 **Joe Ficalora《Quality Function Deployment and Six Sigma》(QFD Handbook, 2nd ed.)**。本书把这套原本用于设计实体产品的路线**迁移到设计 Agent 上——沿用阶段划分，不改方法论归属**。
- **末阶段写法（关键）**：早期个别培训资料曾把末阶段写作「**控制 Control**」，**成熟体系统一用「能力 Capability」**——新产品在量产前要兑现的是**过程能力**，而非对既有流程的控制。**主 skill 与子 skill 中一律写 Capability，禁止回写成 Control。**
- **四阶段一句话**：
  - **Concept（概念定义）**：把「合格」定清楚——must-be → 可量化 CTQ、自治边界、成功判据；
  - **Design（设计）**：先定接口契约 / 数据契约 / 收敛动作空间，让错误动作根本做不出来；
  - **Optimize（优化）**：在冲突目标间显式权衡，拆开化解 + 把可靠度预算集中到瓶颈步；
  - **Capability（能力验证）**：用全程留出、答案独立确认的真值集架**能力门**，达标才放行进入责任链。
- **归属核实记录**：完整归属（SBTI / Zinkgraf / Ficalora QFD Handbook）原文出现在第 11 章 (file15) 第 4 行，已逐字核实；03 / 04 / 27 三文件仅出现「SBTI」一处归属，未出现 Zinkgraf 或 Ficalora。**当前稿不杜撰卷期号与页码。**

## 形象说法 → 标准工程对应物（映射表）

> 术语处理原则（附录 B 原文）：正文允许保留少量形象说法帮助跨背景读者建立直觉，但每个形象说法都**必须在术语表中落回标准工程对应物**，避免把隐喻误作成熟学术分类。

| 形象说法 | 标准工程对应物 | 出处 |
