## 二、十二阶段流水线（P0–P11）

主 skill 用 R≈pⁿ 主线把流程焊成一条**两速控制流水线**：快内环（P1–P6，驾驭工程/执行）
+ 慢外环（P7–P11，改进 + 规模化准入：**DMAIC 判例飞轮 P7–P10**、**生产准入终点 gate P11**），中间靠**独立测量总线**（P7）耦合。每阶段给出：
goal / 对应八审章节（真实章号，**非子 skill 漂移声明**）/ 路由子 skill（或内联）/
内联方法 / 进入&退出门禁 / 交付物。

### ★ 分诊出口（与第 5 章控制档位精神一致，优先于十二阶段）

**不是所有任务都要走完整十二阶段。** 立项分诊第一动作是按问题本性定档位：

- **简单查表 / 算式类（N≈1）** —— 一行确定性代码即可算死的指标（如算 CpK、算成本、
  按口径算经营贡献、按工序号精确命中失效模式库），**直接走确定性代码，不强制走全流水线**。
  这类步骤模型根本不该碰（第 8/12 章「算力归代码」、第 5 章控制档位）。
- **有套路、处理少数几步** —— 指令面契约（强制 Schema）+ 轻校验即可，无需挂慢外环。
- **真复杂诊断（长链条）** —— 才完整运行治理面，必要时挂慢外环 DMAIC。
- **不可逆执行** —— 无论档位，强制人工门（第 5/14 章，与置信度解耦）。

> 风险提醒：主 skill 不得把所有阶段门禁写死为硬 gate，否则与低风险轻量任务的直接执行
> 路径冲突。SKILL.md 必须保留本分诊出口。

### P1　三层控制面设计与任务分诊（Design 总骨架）

- **Goal**：在模型外盖三层物理隔离控制面（指令面 / 信息面 / 运行治理面）+ 写在动手之前的
  任务分诊，作为从认知篇过渡到构建篇、路由各构建 / 治理子 skill 的枢纽。
- **对应八审章节**：第 4 章（三层控制面 + 任务分诊漏斗图 4-4 / 判据表 4-3 / 表 4-2 手段映射，
  file08）；第 11 章（CDOC 四阶段 → 三层控制面映射图 11-4，file15）；第 12 章（分离原理：
  空间 / 时间 / 层次，file16）。
- **路由子 skill**：`agent-control-plane-design`。
- **内联方法**：指令面 = API 契约强制 Schema（让错动作做不出来 B3）；信息面 = Need-to-Know +
  动态折叠（三千词折成一行结论）；运行治理面 = 驾驭工程四步闭环（状态外置 → 权限隔离 →
  强制校验熔断 → 人工审批）；分诊上半场做减法；必要多样性（控制花样 ≥ 任务花样）。
- **进入门禁**：P0 放行凭证已出（控制档位 / 人工门 / 瓶颈步三项结论）。
- **退出门禁**：指令面输出过 Schema 缺字段不放行；信息面写 SQL 只给表结构不给数据行；
  不可信外部内容隔进数据区；状态外置 JSON 锁字典；权限白名单；季度护栏审计（触发率为零 /
  误报率过高标"待评审"防护栏膨胀）。
- **交付物**：`control-plane-spec.md`（指令面 Schema 契约 + 信息面可见性矩阵与折叠规则 +
  运行治理面状态字典模板与审批网关）+ 任务分诊漏斗判据表 + CDOC 四阶段设计文档**初稿**（最终定稿归 **P5 模式 C**·owner=`enterprise-agent-assessment`；P1 仅出上游初稿，含 Concept 的

### P2　构建·外部接入与信息控制面（快内环输入治理）

- **Goal**：为快内环接上受控外部能力（MCP 读写分治）+ 把每个检索查询前置档位判据，
  把脏证据洗成带置信度的干净输入。
- **对应八审章节**：第 6 章（MCP / 函数调用三类失效 / 读写分治 / N×M→N+M，file10）；
  第 7 章（Agentic RAG 召回→过滤→排序→消解→熔断 / 分级检索表 7-2 / 信用分三维，file11）。
- **路由子 skill**：`agent-control-plane-design`（接入层）+ `agent-guardrails-config`（信息洗净 / 熔断）。
- **内联方法**：MCP = USB-C 接口塌缩集成；函数调用只输出结构化意图、代码确定性执行；
  **先算数学（SQL 硬指标一票否决）后比语义**；引文信用分三维（信源权威 / 时效衰减 /
  交叉验证）；结构化地说"我不知道" `{"status":"knowledge_insufficient","missing_info":"…"}`。
- **进入门禁**：P1 控制面骨架与读写分治边界已定。
- **退出门禁**：网关参数校验拦下未注册参数 / 未在枚举集的值；入库强制版本号 + 有效期、
  检索只返当前有效版（防时间碰撞）；跨域向量空间物理或逻辑隔离 + 校验请求者权限令牌；
  检索置信度门控低于阈值不注入、强制返回"知识不足"；外部来源裹 untrusted、块内指令不执行；
  高风险场景不能同一模型既生成又裁决证据。
- **交付物**：外部能力地图（顾问型 / 执行型白名单）+ MCP 服务器注册清单 + MCP 兼容性矩阵
  （表 6-3）+ 信息控制面流水线规格（召回 / 过滤 / 排序 / 消解 / 熔断）+ 分级检索分诊表（表 7-2）。

### P3　构建·代码沙盒计算控制面（快内环计算治理）

- **Goal**：把确定性计算物理剥离给代码沙盒，确保每个数值可复验、自愈安全
  （算力归代码，解释归模型）。
- **对应八审章节**：第 8 章（代码沙盒 / 计算下推 / 三层隔离 / 代码验证层 / 幂等自愈 /
  确定步随机步切分表 8-1，file12）。
- **路由子 skill**：`agent-control-plane-design`。
- **内联方法**：算力归代码、解释归模型；CpK 等有唯一公式的指标必由确定性代码算（如
  CpK=0.82，是代码算的、可一行不改重跑，"严重偏移、主因主轴"才是模型判断）；计算下推
  大聚合给数据仓库；三层隔离（网络断网 / 文件系统只读挂载 / 资源 CPU·内存·磁盘硬上限，
  如 60 秒 / 2GB / 500MB，触顶即杀）；自愈 3–5 次 + 三档降级；纯计算与带副作用代码物理分离。
- **进入门禁**：P2 干净输入已就位（沙盒外部数据预先只读注入）。
- **退出门禁**：数值结果必须可复现、不停留在模型自由生成层；代码验证层（Schema 绑定 /
  AST 静态扫描 / 库函数白名单 + 危险调用拦截 / 断言 / 黄金样例 / 单元测试 / 差分校验）；
  大文件入口静态拦截（如 >500MB）；读写隔离只在受版本控制的沙盒数据分支；带副作用动作
  必须幂等（唯一业务键服务端去重，或挪出自愈循环单独提交）。
- **交付物**：代码沙盒安全配置清单（网络 / 文件 / 资源上限 + 计算下推阈值与拦截动作）+
  任务链确定步 / 随机步切分表（表 8-1）+ 幂等核查清单。

### P4　构建·多智能体与编排（快内环组织设计 + 焊成流水线）

- **Goal**：把已验证可靠的单 Agent 链沿权责边界切成数字科层制（谁生成 / 复核 / 批准 /
  执行），再用 DAG / 状态机焊成可恢复、可审计、可治理的执行流水线；同时是慢外环的进料口
  （生成追踪 ID 与状态轨迹）。
- **对应八审章节**：第 9 章（数字科层制 / 生成者≠审查者 / 拆分三道判据 / 沟通税 /
  共因失效 / 安灯 / 契约注册表，file13）；第 10 章（群聊→DAG / 两类失败两套管辖 /
  条件分支 waitForApproval / 失败常态化三件套 / 就近校验 / 选型表 10-6，file14）。
- **路由子 skill**：`agent-human-loop-design`（科层制制衡与节点闭环）+
  `agent-control-plane-design`（编排引擎）。
- **内联方法**：默认先别拆、举证责任在拆分方；按技能正交切、不按界面菜单切；生成者与
  审查者物理隔离、用**不同知识库**找反例；DAG 拓扑写死流转；两类失败分管（业务错回模型
  自愈 / 物理错引擎带抖动指数退避 / 未知错进死信队列告警）；幂等键由引擎生成、模型零感知。
- **进入门禁**：P3 单 Agent 链已被验证可靠（拆分前提）。
- **退出门禁**：拆分收益须盖过沟通税；每个 Agent 独立上下文窗口、只传 JSON 契约；契约带
  版本号、不兼容即熔断；冗余须真独立（同模型同库三遍 = 伪冗余 / 共因失效）；不可逆动作
  waitForApproval 物理挂起；"超时但可能已落库"先只读探测再决定重试；每节点紧后挂独立校验门。
- **交付物**：Agent 拆分决策表（表 9-1）+ 生成者—审查者制衡设计规范（附共因失效自查表 9-3）
  + 通信契约 Schema 注册表 + 数字科层制拓扑图（图 9-3）+ DAG / 状态机定义 + 失败处理矩阵
  （表 10-4）+ 生产级 JSON 状态字典模板（`checkpoint_hash` / `idempotency_key` /
  `schema_version`）+ 编排引擎选型表（表 10-6）。

### P5　设计期质量内建复核（CDOC Capability 预演 / 可选回看）

- **Goal**：在装护栏前用 CDOC 把质量造进图纸：must-be → 可量化 CTQ、收敛动作空间、显式权衡、
  上线前能力门；用 TRIZ 分离原理化解"自治 vs 可验证"假矛盾而非折中。
- **对应八审章节**：第 11 章（CDOC 四阶段 Concept / Design / Optimize / Capability /

### P7　独立测量总线接通（慢外环地基 Measure）

## 四、术语 canon 钉死项（流水线转述时不得简化）

- **R≈pⁿ** 真实写法是**条件概率连乘 `R = ∏ p_i`**（`p_i = P(step 正确 | 上游状态 / 上下文
  长度 / 噪声 / 既有错误)`），**pⁿ 只是便于建立直觉的乐观基线**，真实衰减更陡
  （self-conditioning 误差传播）。`0.95²⁰ ≈ 0.36`。**不可简化成纯 pⁿ。**
- **CDOC 末阶段是 Capability（能力）不是 Control**。
- **三道防线 = 第 13 / 14 / 15 章**，章标题用"第一/二/三**层**防线"；P6 三道防线锚定真实
  章号，不照搬已漂移的子 skill 声明。
- **人工门 = f(可逆性)、与置信度彻底解耦**（能撤放手做、不能撤先问，高置信但撤不回照样先问）。
- 参考文献只引附录 A 与 file28 已核验字段（arXiv 号 / DOI / 年份 / 出版社），**当前稿不
  杜撰任何卷期号与页码**。

## 五、P5 能力门 vs P11 影子运行（两道门，别混）—— [测试修复 BUG-2]
两处都在"上线前证能力"，但**不是同一道门、不重叠**：
- **P5 CDOC 能力门**（设计期 / 离线）：用**留出的历史真值集**一次性证"造对了"——根因命中率/逃逸率达红线才准进入构建后段；oracle 先于实现；**不达标 = 回设计**。
- **P11 影子运行**（上线期 / 在线）：接**真实生产数据流、输出不执行**，1–2 周证"真实分布上仍达标"——能抓出离线真值集不会出现的偏差（如 8D 夜班来料缺失）；**不达标 = 不放流量**。
- 一句话：P5 证"设计期能力"(离线、自留题)，P11 证"真实分布能力"(在线、真数据)；P5 过了才配进 P11，P11 过了才配放量。

## 六、阶段交付物契约（最小 schema，吃自己"指令面契约"的狗粮）—— [测试修复 BUG-3]
主 skill 不靠"交付物名对上"接力；每阶段产物须满足下表**最小字段**，下阶段**消费前校验**字段齐全（缺字段即按 P1 指令面"缺字段不放行"处理）：

| 阶段产物 | 最小必备字段 | 下阶段消费校验 |
|---|---|---|
| P0 assessment-report | verdict∈{GO,CONDITIONAL_GO,NO_GO}·作业链N·不可逆动作清单·瓶颈步·容差带 | P1 读控制档位/不可逆清单；非 GO 不进 P1 |
| P1 control-plane-spec | 指令面Schema·信息面可见性矩阵·状态锁字典·分诊档位 | P2–P4 按状态锁字典推进；缺锁项不放行 |
| P5 CDOC 文档 | CTQ表(可测+红线)·动作空间收敛·**能力门真值集+红线** | P6/P11 读能力门红线作准入条件 |
| P6 护栏配置 | 三层护栏·风险分级A/B/C·**不可逆动作→可逆门清单**·判例库Schema | P7+ 读判例库Schema；可逆门清单进 P11 |
| P7 测量规范 | 测量维度(Y)·独立审查机制·反例评估集·trace规范 | P8–P10 以此为唯一计分依据；非独立测量不采信 |
| P11 准入报告 | verdict∈{READY,CONDITIONAL,BLOCKED}·五步门结果·三组看板指标 | 收口；BLOCKED 不上线 |

> 把书第 4 章「指令面=API 契约、缺字段不放行」用到主 skill 自身的阶段接力上——阶段间是**机械契约校验**，不是"看名字接"。
>
> **裁决枚举规范化（机械校验口径）—— [v2 修复 BUG-枚举漂移]**：子 skill 报告里的人类可读裁决（如 P0 的 "CONDITIONAL GO"、P11 的 "CONDITIONAL"）在机械校验前先规范化为 token——`trim → 大写 → 空格/连字符转下划线`，故人类可读的 "CONDITIONAL GO" ≡ 契约 token `CONDITIONAL_GO`（消除 `COND` 缩写歧义）。两套裁决枚举**不可混用**：P0 立项门出 `{GO, CONDITIONAL_GO, NO_GO}`（owner=`enterprise-agent-assessment`），P11 准入门出 `{READY, CONDITIONAL, BLOCKED}`（owner=`agent-production-gate`）；下阶段按本规范化口径做相等校验，不靠字面串直接比对。子 skill 报告正文可保留中文友好裁决（如"CONDITIONAL GO"），但须在裁决处括注规范 token 供机械接力。
