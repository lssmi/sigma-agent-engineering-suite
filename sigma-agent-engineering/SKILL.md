# Sigma Engineering：企业 AI Agent 主编排器

> **一句话定位**：`sigma-agent-engineering` 是全书唯一的**端到端主编排器**——它用质量工程这门学科去建造和治理 Agent 本身（而非让 Agent 去跑六西格玛）。它**不取代** 6 个子 skill，而是用主线 **R≈pⁿ** 把它们焊成一条**两速控制流水线**（快内环=驾驭工程/执行，慢外环=DMAIC/改进，中间靠独立测量总线耦合），并在每个阶段判断"内联本书方法即可"还是"路由到某个子 skill 深做"。

主 skill 自己持有四份共享底稿（术语 canon、阶段流水线、方法论账本、两案例脚本），保证 6 个子 skill 引用同一套口径。本 SKILL.md 只做导航与分流；任何阶段的细节都下沉到 `reference/`（渐进式披露）。

## 1. 主线速览（详见 `reference/spine.md`）

四个概念是整条流水线的方法论锚，转述时**不得简化**：

- **R≈pⁿ（主线）**：任务整体可靠度 ≈ 单步可靠度的步数次方。真实写法是**条件概率连乘 `R = ∏ pᵢ`**（`pᵢ = P(step_i 正确 | 上游状态 / 上下文长度 / 噪声 / 既有错误)`），`pⁿ` **只是便于建立直觉的乐观基线**，真实衰减因 self-conditioning 误差传播更陡，**绝不可简化成纯 pⁿ**。基准直觉 `0.95²⁰ ≈ 0.36`：每步 95%、跑 20 步，整体只剩约三分之一。

- **两速控制系统**：**快内环**（执行=驾驭工程，毫秒到秒，压方差、每步过门禁才放行，不学习不反思）+ **慢外环**（改进=DMAIC 五阶段跨周期，把真实失败固化为永久门禁）+ 中间**独立测量总线**耦合两环（把 trace 采集评估换算成逃逸率，测量必须独立于被测对象）+ **C→D 判例飞轮**把每次修复固化为带不走的资产。大多数团队只装了快内环，于是停在"Demo 能跑、生产会崩、崩了无从定位"。

- **三手段（你能对 pⁿ 做的全部操作只有三种）**：① 抬高单步可靠度 base p（关键是**降单步方差**而非抬均值）；② 缩短步数 N（删不增值随机步、把确定逻辑剥给代码）；③ 给失败兜底封顶 ⊓（Safety-I 拦可预见失效 + Safety-II 优雅降级，前置条件=状态外置 + 重试幂等）。

- **正交轴 II 信任与遏制**：对付对抗性（提示词注入 / 越权）的**独立轴**，**不是 pⁿ 随机误差**（把对抗性硬塞进 pⁿ 是错的）。三原则：外部内容当数据非指令；别让一个 Agent 同时握私有数据 + 读不可信内容 + 对外发送（**致命三联**）；最小权限并在网关机械执行。

> CDOC 末阶段是 **Capability（能力验证）不是 Control**；DMAIC 的末阶段才是 Control。两者不可混（详见 `reference/terminology.md`）。

## 3. 分诊出口（关键：不是所有任务都要走完整十二阶段）

立项分诊第一动作是**按问题本性定控制档位**（与第 5 章控制档位、第 8/12 章"算力归代码"精神一致，**优先于十二阶段**）：

- **简单查表 / 算式类（N≈1）** —— 一行确定性代码即可算死的指标（算 CpK、算成本、按口径算经营贡献、按工序号精确命中失效模式库）。这类步骤模型根本不该碰：**直接走确定性代码，不强制走全流水线**。
- **有套路、处理少数几步** —— 指令面契约（强制 Schema）+ 轻校验即可，无需挂慢外环。
- **真复杂诊断（长链条）** —— 才完整运行治理面，必要时挂慢外环 DMAIC。
- **不可逆执行** —— 无论档位，强制人工门（与置信度彻底解耦：能撤放手做、不能撤先问，高置信但撤不回照样先问）。

> **轻量 8D 快车道（明确放行，免走全程）**：单批次 CpK 超差 / 查表式核对 / 按工序号命中 FMEA 这类 **N≈1** 任务，直接确定性代码出结果即可，不要起 P0–P11。中间档（有套路的几步）走 **P0 立项 + P1 指令面契约 + P6 可逆门**（不可逆动作仍挂人工），跳过慢外环 P7–P11。只有"跨工序、多源交叉、根因发散"的真复杂诊断（如 8D 案例的 ⑤⑥ 交叉比对推根因）才值得走全十二阶段。**判据 = 任务的 N + 是否含不可逆动作，而非默认走全程。**

**已明确点名某个子 skill 的局部任务** —— 用户已经知道要做哪一段（如"帮我配护栏""设计审批流"），直接用对应子 skill，不必从 P0 起编排。主 skill 只在"把零散环节串成一条可走流水线"或"不知道该用哪个子 skill"时才接管。

> 风险提醒：主 skill 不得把所有阶段门禁写死为硬 gate，否则与低风险轻量任务的直接执行路径冲突。本分诊出口永远优先。

## 3.5 定位：这套方法论在你现有栈的哪一层（治理层，非编排框架替代品）

> **named pattern·本地扩展**：下表是本套件的定位说明（**非书内对比表**）；所列框架名为生态举例、非背书或对标。

> 常见误解：把本套件当成又一个 Agent 编排框架来选型。**它不是**——它是焊在你**已有框架**之上的治理层。带着你的 LangChain / CrewAI / AutoGen / LangGraph 来，把三层控制面 / 三道防线 / 独立测量总线 / 判例飞轮焊在框架产出的 trace 与节点上。

| 维度 | LangChain / CrewAI / AutoGen / LangGraph 等编排框架 | 本套件（Sigma Engineering 治理层） |
|---|---|---|
| 解决的问题 | 怎么把 LLM / 工具 / 多 Agent **接起来跑通**（connect & run） | 跑通之后怎么**抬可靠度下限、可审计、运行中持续自我迭代**（govern & raise R） |
| 抽象层 | 执行平面（快内环的实现载体） | 控制平面 + 慢外环（P7–P10 DMAIC 判例飞轮 + P11 准入）——与框架**正交**、互补 |
| 落地关系 | — | **不是二选一**：本套件不要求换框架，只把治理结构焊在框架产出的节点 / trace 上 |

- 正交性现成证据：`agent-control-plane-design` 把 LangGraph 等列为**可插入实现**；`reference/case-ecom.md` 记录"平台并非按本书框架设计，却长出高度一致的工程原则"——治理层与具体实现解耦。
- 极端例证（dogfooding）：`tablygo` 是一个**无任何编排框架的纯统计 Web 应用**，仍可装上数值引用门 + 有界 store + 诚实标签——进一步证明本套件是与框架解耦的治理层，而非框架替代品（引用为**本地实例、非本套件 skill**）。

## 5. 两条贯穿案例（端到端示范）

两案例骨架完全一致——**独立、看真产物、找反例、自标定**；一个对付随机误差，一个顶住恶意指令，背后同一套 Sigma Engineering。

- **制造业 8D 诊断 Agent**（→ `reference/case-8d.md`）：九步骤约二十步链（MES 取数→代码算 CpK 判异→检索 FMEA 库→调历史 8D→交叉比对保养/来料/环境→根因假设→起草 8D 报告→冻结物料→挂签字）沿 P0–P11 走完整圈，`0.95²⁰≈0.36` 锚主线、约束硬化 58%→78%、夜班来料字段缺失沿 DMAIC 转一圈。主要对付**随机误差**。

- **电商只读经营决策平台**（→ `reference/case-ecom.md`）：同一套范式对付随机误差 + 对抗性，**全程只读零写回**；北极星=概念性经营贡献（非真实利润、跨期近似显式标 estimated）；外部报表口径熔断 + 裹 untrusted；安慰剂对照组专找"其实没用"的反例。除随机误差外还要**顶住恶意指令**。
